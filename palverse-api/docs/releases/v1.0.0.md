# Palverse API v1.0.0

## Release Summary
This is the first stable release of the Palverse API (v1.0.0). It provides the complete foundational backend required to support both the Next.js Web Dashboard and the Flutter Mobile Application. It utilizes Bearer token authentication via Laravel Sanctum to manage Public, Merchant, and Admin access levels.

## Included Modules
- Authentication & Session Management
- Role-based Access Control (RBAC)
- Public Reference Data (Categories, Cities, Zones, Subscription Plans)
- Merchant Store Management (Info, Media, Offers, Working Hours, Social Links, Subscriptions)
- System Notifications (Database/Mail)
- Administrative Overrides & Moderation
- Static Pages & FAQs
- System Settings & Application Bootstrap

## Security Highlights
- Strict validation envelopes for all HTTP exceptions.
- Hardened upload validations (preventing SVG XSS).
- Session revocation capabilities.
- Audit logs for administrative actions.
- Automatic cache invalidation.

## Operations
- Fully configured for asynchronous Queues (Redis).
- Cron-ready scheduling for subscriptions.
- Readiness and Health probe endpoints configured for load balancer/Kubernetes compatibility.
- Deployment automation shell script included.

## Upgrade Notes
When deploying this version to production for the first time:
1. Copy `.env.production.example` to `.env`.
2. Do **not** run the `DemoDataSeeder`. Run `php artisan db:seed --class=ProductionSeeder --force`.
3. Create your first admin using `php artisan palverse:create-admin`.
4. Ensure the Supervisor worker and Cron jobs are active.
